Design Guide for RiskyDE

     If you'd like to develop an application for RiskyDE, I recommend you follow the guidelines listed below for user interface (UI) consistency. This guide will also get you set up from writing an app from scratch.

While I continuously update RiskyDE, please check back at this page, as I will tweak things and add features in the future.

1. File Creation

     First and most importantly, thank you for your interest in developing an app for RiskyDE. I find developing RiskyDE to be a rewarding experience. Hopefully you will, too.

     To begin, you must create an .html file named after the app you wish to design. This file goes into the '~/sub/apps' directory1. Along with this file, create a folder, also named after the app, and place it in the same directory as the .html file2. You will use this folder for storing resources for the app, such as javascript and graphics. If you choose to do so, you can copy the contents of the template.html file found within the '~/sub/extras' directory.

This file is available for the convenience of developers. It contains the barebones of a RiskyDE application, including the menubar, close button, and the main container where windows are displayed. If you don't wish to copy the file, take a look at the contents. It shows the proper syntax for a RiskyDE app.

     Once you have the contents of template.html copied over, you'll need to name the application window. This name must be typed within the 'title' field, where null is. This will help the user identify which window is opened in RiskyDE.


1. The root directory of RiskyDE is marked as '~' in this documentation.

2. This directory is known as an appdata folder. You should place the license for your app in the root of this folder.

2. Basic Window Design

     While creating your application, keep in mind that RiskyDE is designed to only run one application at a time. This is due to the very low resolution RiskyDE utilizes, which is SVGA, or 800x600 pixels. Also, consider leaving space above or below your application window for a small dialog box, if the application requires it. At this time, RiskyDE does not support overlapping dialog boxes.

     Now, let's think about the application we wish to create. It is recommended to always have a title bar, excluding dialog boxes. Will your application require a detail bar? Let's look at the code that makes up the design of a standard window.

Class name: windowdiv
Purpose: Required. Removes any margin or padding automatically added to containers with this class.

Class name: wincontent
Purpose: Required. Sets background color, border and overflow values of the window.

Class name: titlebar
Purpose: Recommended. Sets font values for the name of the window.

Class name: detailbar
Purpose: Optional. Sets font values and border for the portion below the titlebar.

     While these classes already do some of the work for us, we will need to set certain values, such as the width and height of the window. Every developer is different; we all have ways that we express ourselves through code. I recommend you look at the source for any application included in RiskyDE, and you'll see how I format the windows.

     The following recommendations are important for maintaining consistency and ease of use.

1. Always place important UI controls in obvious locations.

2. Never drastically change the way the user interacts with RiskyDE.

3. If your app requires a help file, try to write the documentation to be as clear as possible.

3. Setting Up the Director

     The way users launch apps built for RiskyDE is through the Chooser. The Chooser is a dialog box that holds a Director, or a link to each app. Each Director must have a unique icon and label. The Director icon for your app should be stored in its appdata folder. You can then assign a Director entry in the Chooser that links to the icon.

     To add a Director to the Chooser, you need to create a figure element, and assign the class appicon to it. Next, add the onclick attribute after the declared class, and add this line in-between the quotation marks: window.location.replace('apps/REPLACE_ME.html')3. This short javascript command redirects to the app without leaving history on the user's browser. This is an intentional design choice of RiskyDE.


3. Be sure to replace the text 'REPLACE_ME' with the name of your app's html page.
